Some commands from Vim and linux

Posted on 2018-12-09

Some commands

Life is easier with:

####VIM
Not edit mode:

  1. ctrl+f: like pagedown
    ctrl+b: like pageup
  2. 0 or Home: move to the beginning of this line
    $ or End: move to the end of this line
  3. G: move to the last line of this doc
    gg: move to the first line of thie doc
  4. n: move n lines after
  5. :1,$s/word1/word2/g: between first and last line, replace word1 with word2
    :1,$s/word1/word2/gc: need to be confirm before replacement
  6. x: del
  7. yy: copy current line
    p, P: paste in next/previous line
  8. u: recover from the previous actio
    Ctrl+r: redo the previous action
    .: 重复前一个动作

In edit mode:

  1. ZZ: 没有更改，不存储就离开；有更改就存储后离开

####Using Linux for sure:

  1. Ctrl+a: move to the beginning of the command
    Ctrl+e: move to the end

Uber Chart

Posted on 2018-12-09

Umbrella Helm Chart

在Uber chart中, values的覆盖

Many of the charts in the official charts repository are building blocks for creating more advanced applications. But charts may be used to create instances of large-scale applications. In such cases, a single umbrella chart may have multiple subcharts, each of which functions as piece of the whole.

The current best practice for composing a complex application from discrete part is to create a top-level umbrella chart that exposes the global configuration, and then use the charts/ subdir to embed each of the components.

But sometimes, it is a little bit confusing on how to overwrite a child’s config from uber global values.yaml.

Like for ckey chart, cmdb is enabled by default from cmdb.enabled to true in parent’s values.yaml, but when I create the uber chart in which ckey is child, giving ckey.cmdb.enabled to false would not help me to disable cmdb in ckey. Rather, cmdb would be created anyway, giving a confliction with cmdb which is a real child from the parent(secrets creation).
